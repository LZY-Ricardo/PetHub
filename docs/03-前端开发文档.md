# 宠物管理与服务平台 - 前端开发文档

## 1. 技术栈与架构设计

### 1.1 技术选型

| 技术 | 版本 | 用途 |
|-----|------|-----|
| React | 18.x | 前端框架 |
| Ant Design | 5.x | UI组件库 |
| React Router | v6 | 路由管理 |
| Fetch API | 浏览器内置 | HTTP请求 |
| Vite | 5.x | 构建工具 |
| Day.js | latest | 日期处理 |

### 1.2 架构设计

#### 1.2.1 整体架构
```
Browser
  -> React + React Router + Ant Design
  -> AuthContext（登录态）
  -> 页面组件（Home/Pet/Forum/LostPet/Admin/User）
  -> fetch('/api/*') 请求后端
  -> Vite dev proxy 转发到 http://localhost:3000
```

#### 1.2.2 分层说明
- **页面组件**：完整的页面级组件，对应路由
- **布局组件**：主布局、路由守卫
- **状态层**：`AuthContext` 维护用户与 token 信息
- **请求层**：各页面直接用 `fetch` 调用后端 API

### 5.2 路由守卫

#### ProtectedRoute（登录守卫）
```javascript
const ProtectedRoute = ({ children }) => {
  const { user } = useAuth();

  if (!user) {
    return <Navigate to="/login" replace />;
  }

  return children;
};
```

#### AdminRoute（管理员守卫）
```javascript
const AdminRoute = ({ children }) => {
  const { user } = useAuth();

  if (!user || user.role !== 'admin') {
    return <Navigate to="/" replace />;
  }

  return children;
};
```

## 7. API请求约定

当前前端采用浏览器原生 `fetch`，未引入 Axios 拦截器层。

### 7.1 调用模式
- 公开接口：直接 `fetch('/api/...')`
- 需鉴权接口：在 `headers` 中手动添加 `Authorization: Bearer {token}`
- 统一解析后端返回结构：`{ code, message, data, timestamp }`

### 7.2 示例
```javascript
const token = localStorage.getItem('token');

const response = await fetch('/api/auth/user', {
  headers: {
    Authorization: `Bearer ${token}`
  }
});

const result = await response.json();
if (result.code === 200) {
  // 使用 result.data
}
```

## 10. 构建与部署

### 10.1 构建配置
**文件路径**：`vite.config.js`

```javascript
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import path from 'path';

export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, 'src')
    }
  },
  server: {
    port: 5173,
    proxy: {
      '/api': {
        target: 'http://localhost:3000',
        changeOrigin: true
      }
    }
  },
  build: {
    outDir: 'dist',
    assetsDir: 'assets',
    sourcemap: false,
    chunkSizeWarningLimit: 1000,
    rollupOptions: {
      output: {
        manualChunks: {
          'react-vendor': ['react', 'react-dom', 'react-router-dom'],
          'antd-vendor': ['antd', '@ant-design/icons']
        }
      }
    }
  }
});
```

### 10.2 部署步骤

1. **构建项目**
```bash
npm run build
```

2. **部署到Nginx**
```bash
# 将dist目录上传到服务器
scp -r dist/* user@server:/var/www/pet-platform-frontend/
```

3. **配置Nginx**
```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /var/www/pet-platform-frontend;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```
