## 3. 数据库设计

### 3.1 数据库配置

**数据库名**：pet_management_platform
**字符集**：utf8mb4
**排序规则**：utf8mb4_unicode_ci

### 3.2 数据表设计

#### 3.2.1 用户表（sys_user）

```sql
CREATE TABLE `sys_user` (
  `id` INT PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID',
  `username` VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
  `password` VARCHAR(255) NOT NULL COMMENT '密码（bcrypt加密）',
  `nickname` VARCHAR(50) NOT NULL COMMENT '昵称',
  `avatar` VARCHAR(255) DEFAULT NULL COMMENT '头像URL',
  `role` ENUM('user', 'admin') DEFAULT 'user' COMMENT '角色',
  `contact_info` VARCHAR(100) DEFAULT NULL COMMENT '联系方式',
  `status` TINYINT DEFAULT 1 COMMENT '状态：1-正常，0-禁用',
  `created_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  INDEX idx_username (`username`),
  INDEX idx_role (`role`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

**字段说明**：
- id：主键，自增
- username：唯一用户名，用于登录
- password：密码，使用bcrypt加密存储
- nickname：用户昵称，用于展示
- avatar：头像URL，可为空
- role：用户角色，user（普通用户）或admin（管理员）
- contact_info：联系方式，手机号或邮箱
- status：账号状态，1正常，0禁用
- created_at：账号创建时间
- updated_at：最后更新时间

#### 3.2.2 宠物信息表（pet_info）

```sql
CREATE TABLE `pet_info` (

  `health_status` ENUM('good', 'fair', 'poor') DEFAULT 'good' COMMENT '健康状况',
  `personality` VARCHAR(200) DEFAULT NULL COMMENT '性格特点',

  `adopted_by` INT DEFAULT NULL COMMENT '领养人ID',
  `adopted_at` TIMESTAMP NULL COMMENT '领养时间',
```

- vaccination：疫苗接种情况
- sterilized：是否绝育，1是0否
- status：状态，available（待领养）、pending（申请中）、adopted（已领养）
- photos：照片URL数组，JSON格式
- remarks：其他备注信息
- created_by：创建人ID（管理员）
- adopted_by：领养人ID
- adopted_at：领养时间
- created_at：创建时间
- updated_at：更新时间

## 4. 核心模块实现

### 4.1 用户模块

#### 4.1.1 用户注册
**流程**：
1. 验证用户名是否已存在
2. 验证输入参数（用户名、密码、联系方式）
3. 使用bcrypt加密密码（salt rounds: 10）
4. 插入数据库，角色默认为'user'
5. 生成JWT token
6. 返回用户信息（不包含密码）

**关键代码**：
```javascript
// 密码加密
const hashedPassword = await bcrypt.hash(password, 10);

// 生成token
const token = jwt.sign({ userId: user.id, role: user.role }, SECRET, { expiresIn: '24h' });
```

#### 4.1.2 用户登录
**流程**：
1. 根据用户名查询用户
2. 验证密码（bcrypt.compare）
3. 检查账号状态
4. 生成JWT token
5. 返回用户信息和token

#### 4.1.3 获取用户信息
**流程**：
1. 从JWT中解析userId
2. 查询用户信息
3. 返回用户数据（不包含密码）

#### 4.1.4 更新用户信息
**流程**：
1. 验证用户身份（JWT）
2. 验证输入参数
3. 更新数据库
4. 返回更新后的用户信息

#### 4.1.5 头像上传
**流程**：
1. 使用multer中间件处理文件上传
2. 验证文件类型和大小
3. 生成唯一文件名
4. 保存到uploads/avatar目录
5. 更新数据库中的avatar字段
6. 返回新的头像URL

### 4.2 宠物档案模块

#### 4.2.1 添加宠物（管理员）
**流程**：
1. 验证管理员权限
2. 验证输入参数
3. 处理图片上传（1-5张）
4. 将图片URL数组存入JSON字段
5. 插入数据库
6. 返回宠物信息

#### 4.2.2 获取宠物列表
**查询参数**：
- page：页码，默认1
- pageSize：每页数量，默认10
- breed：品种筛选
- gender：性别筛选
- minAge、maxAge：年龄范围
- status：状态筛选
- sortBy：排序字段，默认created_at
- sortOrder：排序方向，ASC/DESC，默认DESC

**SQL查询**：
```sql
SELECT p.*, u.username as creator_name
FROM pet_info p
LEFT JOIN sys_user u ON p.created_by = u.id
WHERE 1=1
  AND (? IS NULL OR p.breed = ?)
  AND (? IS NULL OR p.gender = ?)
  AND (? IS NULL OR p.age >= ?)
  AND (? IS NULL OR p.age <= ?)
  AND (? IS NULL OR p.status = ?)
ORDER BY p.${sortBy} ${sortOrder}
LIMIT ? OFFSET ?
```

#### 4.2.3 获取宠物详情
**流程**：
1. 根据ID查询宠物信息
2. 查询创建人信息
3. 如果已领养，查询领养人信息
4. 组装数据返回

#### 4.2.4 更新宠物信息（管理员）
**流程**：
1. 验证管理员权限
2. 检查宠物是否存在
3. 验证状态（已领养不能修改为其他状态）
4. 更新信息
5. 返回更新后的数据

#### 4.2.5 删除宠物（管理员）
**流程**：
1. 验证管理员权限
2. 检查宠物状态（仅待领养可删除）
3. 删除宠物记录
4. 删除相关图片文件
5. 返回成功

### 4.3 领养申请模块

#### 4.3.1 提交申请
**流程**：
1. 验证用户身份
2. 检查宠物状态（仅待领养可申请）
3. 检查是否已有申请（同一用户对同一宠物）
4. 检查用户待审核申请数量（最多3个）
5. 插入申请记录
6. 更新宠物状态为"申请中"
7. 返回申请信息

**事务处理**：
```javascript
const connection = await pool.getConnection();
try {

} finally {
  connection.release();
}
```

## 5. 中间件设计

### 5.1 认证中间件（auth.js）

```javascript
const jwt = require('jsonwebtoken');

const authMiddleware = async (ctx, next) => {
  try {
    // 获取token
    const token = ctx.headers.authorization?.replace('Bearer ', '');

    if (!token) {
      ctx.status = 401;
      ctx.body = { code: 401, message: '未登录或token已过期' };
      return;
    }

    // 验证token
    const decoded = jwt.verify(token, process.env.JWT_SECRET);

    // 将用户信息挂载到ctx.state
    ctx.state.user = {
      userId: decoded.userId,
      role: decoded.role
    };

    await next();
  } catch (error) {
    ctx.status = 401;
    ctx.body = { code: 401, message: 'token验证失败' };
  }
};

// 管理员权限验证
const adminAuthMiddleware = async (ctx, next) => {
  if (ctx.state.user?.role !== 'admin') {
    ctx.status = 403;
    ctx.body = { code: 403, message: '无权限访问' };
    return;
  }
  await next();
};

module.exports = { authMiddleware, adminAuthMiddleware };
```

### 5.2 错误处理中间件（error.js）

```javascript
const errorMiddleware = async (ctx, next) => {
  try {
    await next();
  } catch (error) {
    console.error('Error:', error);

    // 业务错误
    if (error.code) {
      ctx.status = 200;
      ctx.body = {
        code: error.code,
        message: error.message || '操作失败'
      };
      return;
    }

    // 数据库错误
    if (error.code === 'ER_DUP_ENTRY') {
      ctx.status = 200;
      ctx.body = { code: 400, message: '数据已存在' };
      return;
    }

    // 未知错误
    ctx.status = 500;
    ctx.body = { code: 500, message: '服务器内部错误' };
  }
};

module.exports = errorMiddleware;
```

### 5.4 参数验证中间件（validator.js）

```javascript
const validatorMiddleware = (rules) => {
  return async (ctx, next) => {
    const { body, query } = ctx;
    const data = ctx.method === 'GET' ? query : body;
    const errors = [];

    for (const [field, rule] of Object.entries(rules)) {
      const value = data[field];

      // 必填验证
      if (rule.required && !value) {
        errors.push(`${field}不能为空`);
        continue;
      }

      // 类型验证
      if (value && rule.type) {
        if (rule.type === 'string' && typeof value !== 'string') {
          errors.push(`${field}必须是字符串`);
        }
        if (rule.type === 'number' && isNaN(value)) {
          errors.push(`${field}必须是数字`);
        }
      }

      // 长度验证
      if (value && rule.min && value.length < rule.min) {
        errors.push(`${field}长度不能少于${rule.min}`);
      }
      if (value && rule.max && value.length > rule.max) {
        errors.push(`${field}长度不能超过${rule.max}`);
      }
    }

    if (errors.length > 0) {
      ctx.status = 200;
      ctx.body = { code: 400, message: errors.join('; ') };
      return;
    }

    await next();
  };
};

module.exports = validatorMiddleware;
```

## 6. 安全策略

### 6.1 密码安全
- 使用bcrypt加密，salt rounds = 10
- 密码长度限制：6-20字符
- 不在日志中记录密码

### 6.2 JWT安全
- token有效期：24小时
- 存储在HTTP Header中：Authorization: Bearer {token}
- 密钥存储在环境变量中

### 6.3 SQL注入防护
- 使用参数化查询
- 不拼接SQL字符串
- 输入参数验证

### 6.4 XSS防护
- 对用户输入进行转义
- Content-Type: application/json
- 不使用innerHTML渲染用户内容

### 6.5 文件上传安全
- 限制文件类型：jpg、jpeg、png
- 限制文件大小：≤5MB
- 生成唯一文件名
- 存储在非Web根目录

### 6.6 接口访问频率限制
- 使用koa-ratelimit
- 限制：同一IP每分钟最多60次请求
