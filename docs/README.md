### 4. [API接口文档](./04-API接口文档.md)
**文件**：`04-API接口文档.md`

**内容概要**：
- 接口基础信息（Base URL、认证方式）
- 统一响应格式
- 7大模块的完整API定义：
  1. **认证模块**（5个接口）
     - 用户注册/登录
     - 获取/更新用户信息
     - 上传头像

  2. **宠物档案模块**（5个接口）
     - 获取列表/详情
     - 添加/更新/删除宠物（管理员）

  3. **领养申请模块**（4个接口）
     - 提交申请
     - 获取我的申请
     - 获取所有申请（管理员）
     - 审核申请（管理员）

  4. **走失宠物模块**（7个接口）
     - 获取列表/详情
     - 发布/更新/删除走失信息
     - 标记已找到
     - 获取我的发布

  5. **社区互动模块**（8个接口）
     - 获取帖子列表/详情
     - 发布/删除帖子
     - 发表/删除评论
     - 点赞/取消点赞
     - 获取我的内容

  6. **文件上传模块**（1个接口）
     - 通用图片上传

  7. **统计数据模块**（1个接口）
     - 仪表盘数据（管理员）

- 每个接口包含：
  - 请求方法和路径
  - 是否需要登录
  - 请求参数说明
  - 响应示例
  - 错误响应
- 接口调用示例（Axios、cURL）

**适用人群**：前后端开发人员、测试人员

## 数据库表结构概览

| 表名 | 说明 | 主要字段 |
|-----|------|---------|
| sys_user | 用户表 | id, username, password, nickname, avatar, role |
| pet_info | 宠物信息表 | id, name, breed, gender, age, status, photos |
| adoption_application | 领养申请表 | id, user_id, pet_id, reason, status |
| lost_pet | 走失宠物表 | id, user_id, name, location, photos, is_urgent |
| forum_post | 社区帖子表 | id, user_id, title, content, images |
| forum_comment | 评论表 | id, post_id, user_id, content, parent_id |
| forum_like | 点赞表 | id, post_id, user_id |

详细表结构请参考[后端开发文档 - 数据库设计](./02-后端开发文档.md#3-数据库设计)

## 注意事项

1. **技术约束**
   - 必须使用指定的技术栈
   - 后端必须使用原生SQL，不允许使用ORM
   - 前端必须使用Ant Design 5

2. **安全要求**
   - 密码必须使用bcrypt加密
   - 接口必须使用JWT认证
   - 防止SQL注入和XSS攻击

3. **代码规范**
   - 遵循文档中的命名规范
   - 添加必要的注释
   - 保持代码结构清晰

4. **测试要求**
   - 接口测试：所有接口必须测试
   - 功能测试：核心功能必须测试
   - 性能测试：关键接口需要性能测试
